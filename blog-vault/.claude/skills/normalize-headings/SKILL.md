# 見出し・構成の一括統一

対象: `$ARGUMENTS`(ファイルパス、glob、またはフォルダ。未指定なら `03_drafts/**/*.md`)

CLAUDE.md の「見出し・構成規約」「文体規約」に従い、対象記事の見出しと構成を統一する。**本文の主張・事実は変えない。**変えるのは見出し表現・階層・順序・リード文の整形のみ。

## 手順

1. 対象ファイルを列挙する(glob/フォルダ展開)。**修正前に対象一覧を表示**してから着手する。
2. 各ファイルについて以下を点検し、規約違反を抽出:
   - 見出し階層が h2(`##`)→ h3(`###`)のみか。h1(`#`)はタイトル1個まで、h4以降は使っていないか。
   - h2 の語尾が記事内で統一されているか(名詞句で終える)。
   - 構成順が「リード → 本論 → まとめ」になっているか。リードは3〜5文か。
   - 見出しに煽り表現・禁止語(CLAUDE.md「使わない表現」)が混ざっていないか。
3. ファイルごとに「変更前 → 変更後」の差分案を提示する。
4. 承認後に Edit で適用する。事実・本文ロジックには触れない。
5. 全ファイル完了後、統一サマリ(各ファイルの主な変更点)を1つにまとめて返す。

## 注意
- 複数記事を横断する重い走査は、読み取り中心ならサブエージェント(Explore)に委譲してコンテキストを節約してよい。
- 見出しと連動する目次・内部リンクがある場合は、見出し変更後にリンク切れがないか確認する。
- 1記事でも判断に迷う構成変更があれば、自動修正せず質問する。
