# このVaultの使い方

同人音声レビューブログの **記事量産パイプライン** をObsidianで回すためのVaultです。
元データは `同人音声_ニッチ選定.csv`（残しています）。これを記事ノート15枚に展開しました。

## フォルダ構成

```
ブログ制作/
├─ _ダッシュボード.md   … Dataviewで全記事を自動集計（ここを毎日見る）
├─ _使い方.md           … このファイル
├─ 同人音声_ニッチ選定.csv … 元データ（参照用に保管）
└─ 記事/                … 1ニッチ=1ノート（現在15枚）
```

## 最初の設定（1回だけ）

1. Obsidianでこの `ブログ制作` フォルダを **Vaultとして開く**（保管庫を開く → このフォルダを選択）
2. 設定 → コミュニティプラグイン → 制限モードOFF → **Dataview** をインストールして有効化
3. `_ダッシュボード.md` を開く → 表が自動表示されれば完了

## 日々の運用フロー

1. `_ダッシュボード.md` の優先度ランキングを見て、書く記事を決める
2. 該当の記事ノートを開き、**構成案チェックリスト**を上から潰す
3. ステータスを進める：`アイデア` → `下書き` → `公開`
   - ノート冒頭のフロントマター `ステータス:` の値を書き換えるだけ
4. 公開したら `公開URL:` にURLを記入
5. ダッシュボードは自動で更新される（手動編集不要）

## フロントマターの項目

| 項目 | 意味 | 書き換える？ |
|---|---|---|
| `優先度スコア` | CSVの合計点 | 基本固定 |
| `ステータス` | アイデア / 下書き / 公開 | **進捗ごとに更新** |
| `狙いキーワード` | 狙うKW | 調査後に追記OK |
| `公開URL` | 公開後のURL | **公開時に記入** |

## 内部リンク（SEOのキモ）

記事の `## 関連` 欄で、関連ニッチを `[[機材別音質比較]]` のように繋ぐと、
グラフビューでサイトのトピッククラスター構造が見えます。
これがそのまま **サイト内の内部リンク設計**になります。
ピラー記事（総合まとめ）を1枚作り、各レビューから繋ぐのがおすすめです。

## 補足

- **ゲーム制作Vaultとは別Vault**で運用してください（性格が違うため混ぜない）。
- このVaultはマークダウンなので、Cowork/Claudeからも今まで通り読み書きできます。
- 公開基盤に育てるなら、マークダウンを静的サイト（Hugo/Astro等）へ書き出す構成も可能です。
