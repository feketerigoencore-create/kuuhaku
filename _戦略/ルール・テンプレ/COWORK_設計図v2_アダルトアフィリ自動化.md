# アダルトアフィリエイト自動化 マスター設計図 v2

## 0. このドキュメントの目的
v1（`COWORK_blog_loop.md`）は「同人音声の記事を1本ずつ作るループ」だった。
v2は対象を**アダルトグッズ（物販）まで拡張**し、**サブエージェント編成で半自動の収益サイトを育てる**ための全体設計図。

---

## 1. 今後やりたいこと（ゴールの言語化）
松岡の意図を整理すると、本質は次の3点：

1. **アフィリエイト収入を“自動で”稼ぐ仕組み化**（手作業を最小化し、放置でも記事が積み上がる）
2. **題材はアダルト系**（既存の同人音声＝作品系 ＋ 新規のアダルトグッズ＝物販系）
3. **「買われたら○％が自分の収益」モデル**（＝成果報酬アフィリ）を軸に、複数案件を貼り分けて収益最大化
4. **これらをサブエージェント（Dispatch）に分担させて回す**

> 重要な前提：これは**合法な成果報酬アフィリエイト事業**。記事は商品の紹介・比較・誘導であり、露骨な性的描写は作らない（構成・紹介・ゾーニングのみ）。

---

## 2. 収益モデル（「○％」の正体）
「買われたら○％」は各ASPの**成果報酬率**。2026年時点の主な案件：

| 案件 | 種別 | 報酬率（目安） | 備考 |
|---|---|---|---|
| FANZA(DMM)通販 | グッズ物販 | 同一商品 **20%** / 同カテゴリ **15%** | 新規会員紹介で +1,050円。動画・同人とも統合運用しやすい |
| NLSアフィリエイト | グッズ物販 | **25%**（高水準） | 国内最大手アダルト通販 |
| ラブトリップ | グッズ物販 | **17%** | 女性向けグッズ中心 |
| 大人のデパート エムズ | グッズ物販 | **12%〜**（実績で上昇） | 実績連動 |
| FANZA同人/動画 | 作品系 | 案件による | v1の同人音声はここ |

→ **戦略**：1記事に複数ASPを貼り、商品・読者層ごとに最適案件を出し分ける（後述MONETIZEエージェントの仕事）。すべて**要審査**・**規約順守**・**年齢ゾーニング必須**。【要検証：最新の報酬率・規約は登録時に各ASP公式で確認】

---

## 3. サイト・アーキテクチャ
### 3-1. 2系統の収益軸
- **作品系（既存）**：同人音声レビュー → FANZA作品アフィリ
- **物販系（新規）**：アダルトグッズレビュー/比較/ランキング → グッズASP

### 3-2. サイト分離 or 統合（推奨）
**推奨＝物販系は別ドメインで新設**。理由：規約リスク分散、ジャンル特化でSEO評価が集中、決済導線（通販）と作品導線の文脈が違うため。
（v1の同人音声サイトはそのまま育成継続）

### 3-3. トピッククラスター構造
各サイト内は「ピラー記事（総合まとめ）＋クラスター（個別レビュー）」で内部リンク設計。
例：ピラー「アダルトグッズおすすめ総合」← 各カテゴリ（ローター/バイブ/オナホ/女性向け…）レビュー記事を内部リンクで束ねる。

### 3-4. データ基盤
- **Obsidian Vault（このフォルダ）＝原稿の母艦**。`01_research → 02_outlines → 03_drafts` のパイプライン
- 完成下書き → **WordPress（アダルトOKサーバー）へ下書き投入** → 人間が公開
- サーバー候補：mixhost（FANZA特典あり）/ ColorfulBox / シン・レンタルサーバー

---

## 4. サブエージェント編成（Dispatch）
オーケストレーター（Cowork本体）が各専門エージェントへ委任する。v1の4体を中核に、物販・収益・公開準備を追加。

| エージェント | 入力 | 出力 | 役割 |
|---|---|---|---|
| **SCOUT**（新） | ジャンル/市場 | ニッチ候補＋優先度スコア | 案件・ニッチ・キーワードの発掘と採点（v1のCSV選定を自動化） |
| **RESEARCH** | ニッチ | `01_research/`調査メモ | 検索意図・KW・競合・差別化軸 |
| **OUTLINE** | research | `02_outlines/`構成案 | H2/H3・タイトル・文字数設計 |
| **PRODUCT-MATCH**（新） | outline | 商品枠＋推奨ASP | 紹介商品の「型」と最適案件の割当（報酬率×読者適合） |
| **DRAFT** | outline(+QA修正) | `03_drafts/`下書き | 本文生成。【リンク挿入】【要検証】【内部リンク】を必ず挿入 |
| **QA** | draft | 合否＋修正指示 | チェックリスト判定（最大3試行ループ） |
| **MONETIZE**（新） | draft | リンク配置案 | どの位置にどのASPを貼るか、CTA設計、新規会員特典訴求 |
| **INTERNAL-LINK**（新） | draft群 | 内部リンク候補 | ピラー⇄クラスターの繋ぎ込み案 |
| **PUBLISH-PREP**（新） | 合格draft | WP投入用整形＋ゾーニングチェック | HTML整形・年齢認証・アイキャッチ枠・メタ記述 |

> 自動化の上限（v1踏襲）＝**WordPress下書き保存まで**。
> 公開・アフィリンク確定・最終内部リンク確定・決済規約確認は**人間（松岡）**。

## 5. 自動化ループ（パイプライン v2）
```
SCOUT(ジャンル) → ニッチ候補N件＋優先度
FOR 記事 in 採用ニッチ:
    research        = RESEARCH(ニッチ)
    outline         = OUTLINE(research)
    productPlan     = PRODUCT-MATCH(outline)      # 商品枠＋ASP割当
    draft           = DRAFT(outline, productPlan)
    試行 = 1
    LOOP:
        result = QA(draft)
        IF 合格: BREAK
        IF 試行>=3: draft に「【要人間確認】」付与 → BREAK
        draft = DRAFT(outline, 修正指示=result.feedback)
        試行 += 1
    draft = MONETIZE(draft)        # リンク配置・CTA
    links = INTERNAL-LINK(draft)   # 内部リンク候補
    ready = PUBLISH-PREP(draft)    # WP整形＋ゾーニング
    保存: 03_drafts/{ニッチ}_{連番}.md（状態＝完成 / 要確認）
    （任意）WordPress下書きとして投入
NEXT
→ サマリ報告：完成本数 / 要確認本数 / 想定収益インパクト
```

## 6. QA合格基準（v2拡張）
v1の基準に物販向け項目を追加：
- タイトル・見出しにキーワードが入っている
- 各紹介セクションに【リンク挿入】がある／**ASP割当が明記**されている
- 不確定な事実（価格・スペック・在庫・報酬率）に【要検証】が付いている
- 【内部リンク】候補（ピラー⇄クラスター）が示されている
- **年齢ゾーニング/CTAの想定が記載**されている
- 露骨な性的描写を含まない（紹介・比較・誘導のみ）
- 文字数・構成が記事タイプ（キュレーション/比較/ランキング）に適合

---

## 7. コンプライアンス・ガードレール（必須）
- **年齢認証（ゾーニング）**を全ページに。アダルトOKサーバー必須（無料ブログ不可）
- 各ASPの**規約・禁止表現・審査**を登録時に確認（【要検証】）
- 景品表示法・薬機法に触れる効能断定をしない（「絶対」「治る」等を避ける）
- 実在価格・スペック・在庫は**人間が最終確認**（自動生成は枠とプレースホルダまで）
- 画像・商品説明の権利（各ASP提供素材の範囲内で使用）

---

## 8. ロードマップ（フェーズ）
- **Phase 0（今）**：v1で同人音声の下書きパイプライン稼働中（試運転1本＝完成）
- **Phase 1**：物販系サーバー＆独自ドメイン契約 → WordPress構築 → 主要ASP登録・審査
- **Phase 2**：SCOUT＋PRODUCT-MATCHを実装し、グッズ系ニッチを10〜20件採点 → ピラー1本＋クラスター5本生成
- **Phase 3**：MONETIZE/INTERNAL-LINK/PUBLISH-PREPを組み込み、WP下書き投入まで自動化
- **Phase 4**：成果データ（クリック/CV/報酬）を見てリライト・案件差し替えループ

---

## 9. KPI・ログ・ダッシュボード
- 生産KPI：完成本数 / 要確認本数 / 試行回数
- 収益KPI：表示回数・クリック・CV・確定報酬（公開後にASP管理画面から）
- ログ：`03_drafts/_生成ログ.md` に1行/記事で追記
- 既存の `_ダッシュボード.md`（Dataview）を物販系にも展開し、優先度×収益見込みで書く順を決める

---

## 10. 意思決定が必要なポイント（次に決めること）
1. 物販系サイトは **別ドメイン新設**でよいか（推奨）／既存と統合か
2. 最初に登録するASPの優先順位（FANZA通販・NLS・ラブトリップ・エムズ）
3. 最初に攻めるグッズ・ニッチ（例：ローター/女性向け/初心者向け 等）
4. サーバー（mixhost 推奨：FANZA特典）

→ この4点が決まれば Phase 1 に着手できる。
