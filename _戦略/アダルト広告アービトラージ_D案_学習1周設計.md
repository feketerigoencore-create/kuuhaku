## 0. 最初に押さえる前提（つまずく前に3点）

1. **このモデル最大の関門は「①ネットワーク開設・入金」。** 案件もLPも後でどうにでもなる。初心者が脱落する場所は決まって「英語の海外サイトで広告アカウントを作り、海外決済で入金する」ここ。だから本書はタスク1に重心を置く。**ここを越えれば1周は回る。**

2. **「広告アカウント（出稿側）」と「アフィリエイトアカウント（オファー側）」は別物。** 2つ作る。
   - 出稿側＝**広告ネットワーク**（HilltopAdsなど）。ここに**お金を入れて**広告を買う。
   - オファー側＝**アフィリエイトプログラム/ネットワーク**（Chaturbate/CrakRevenueなど）。ここから**お金をもらう**。成果（登録）が出ると報酬が乗る。
   - この2つを**クリックID**で繋ぐ仕組みが「ポストバック」（§4で詳述）。

3. **学習1周では「儲け」を期待しない。** $330は**授業料**。CVが1件出れば大成功、0件でも「なぜ0だったか」を言語化できれば成功。利益が出る設計は2周目以降の話。

### 2-2. 推奨オファー【1〜2件】

**1st：Chaturbate の「無料登録 $1（SOI）」。**
- **審査なしで今日から開始**でき、無料登録＝**CV率が最も高い**。最低支払$50で「初CVの手応え」を最短で得られる。学習1周の純粋な題材として最良。
- 単価$1は低いが、**学習目的では「低単価・高CV」がむしろ正解**（CVが出ないと何も学べない）。

**2nd：Stripchat / StripCash の「SOI $3」。** 無料登録のままで$1→$3に単価が上がり、手応え（疑似的な損益）が見えやすい。postback練習もしやすい。

**避ける：** DOI（メール認証の1手間でCV率が落ちる）／PPS・課金成果（固すぎる）／LiveJasminのPPL（承認ゲートで初手に不向き）／BongaCash（$200の最低支払で着金が遠い）。

### 2-3. ポストバック（成果計測）の設定有無

- **Chaturbate / Stripchat(StripCash) / CrakRevenue：いずれもポストバックURL設定とsub-id受け渡しに対応**（[C]）。つまり「成果が出たら広告側に教え返す」ことが正規にできる。
- **CrakRevenue** は `{!subid!}`（あなたが送るID）／`{aff_sub}`（返ってくる枠）を使い、全オファー共通の **Global Postback**（マネージャー有効化が必要）も用意。
- 最短ルートは **Chaturbate を直リンク**（自分で制御するsub-idで送客）。**postbackの仕組みを学ぶ**なら、間に無料トラッカー **BeMob** を噛ませるのがクリーン（§4）。
- 【要確認】Japan からのアフィリ登録自体に明確なブロックは見当たらず（Chaturbate/CrakRev枠は実質オープン）。唯一の不確実点は**支払い**：Paxum・暗号資産は日本から機能するが、**初回登録後にCrakRevenue/各社サポートへ「日本からのPaxum/暗号での出金可否」を一度確認**しておくと事故らない。

## 3. 最小プリランダー/LP【動くコード付き】

### 3-1. 方針：凝らない

学習1周のLPは**「年齢確認ゲート＋送客」だけ**。デザインも訴求も作り込まない。唯一の必須要件は、**広告ネットワークが付けてくるクリックIDを、送客先にそのまま引き継ぐ**こと（これが計測＝postbackの肝）。

### 3-2. 既存LP雛形の流用

- プロジェクト内に**HTMLのLP雛形は未作成**（既存は.mdプランと構成SVGのみ）。ゼロから最小実装した。
- ただし **B案 §4「年齢確認ゲート」の設計思想はそのまま流用**：入口に18禁ゲート／運営者情報・問い合わせ動線／露骨表現を出口まで出さない、という関所の考え方を本LPでも踏襲している。

### 3-3. 納品物：`prelander_age_gate.html`（同フォルダに生成済み）

最小ワイヤー（Claude Codeで作れる粒度）：

```
┌─────────────────────────────┐
│  CAM HUB（無難なロゴ）          │
│                              │
│  あなたは18歳以上ですか？        │
│  Are you 18 or older?        │
│                              │
│  この先は成人向け。年齢確認が必要。  │
│                              │
│  [ はい、18歳以上 → 入場する ]   │ ← クリックIDを引き継いでオファーへ
│  [ いいえ（退出する） ]          │ ← google.com等へ離脱
│                              │
│  ※広告/外部成人サービス/18禁注記   │
└─────────────────────────────┘
```

**仕込んである計測ロジック（ファイル冒頭の「ここだけ書き換える」3項目）**
- `OFFER_URL`：送客先（アフィリリンク または トラッカーURL）に差し替え。
- `INCOMING_CLICK_PARAMS`：着地URLに付くクリックIDのパラメータ名（例 `clickid` / `click_id`）。**実マクロ名は各ネットワークのトラッキング設定で確認【要確認】**。
- `OUTGOING_CLICK_PARAM`：オファー側に渡すID名（CrakRevenue等は `subid`/`aff_sub`系）。

「入場」を押すと、着地URLからクリックIDを拾い → オファーURLに付け直して送客する。**この1ファイルだけで「クリックIDの受け渡し」を実地で学べる。**

## 4. ポストバック/計測の仕組み（1周の心臓部）

文章で一度通しておくと、§5の手順で迷わない。

**postback＝「成果が出たら、オファー側のサーバーが広告側のサーバーに“あのクリックが成約したよ”と電話し返す」仕組み（S2S）。** Cookieに頼らず最も確実。

**繋ぐ糊が「クリックID」。** クリック1回ごとのユニークID。送客時に前へ渡し、成約時に同じIDが返ってくることで「どのクリックが成果になったか」を一致させる。

```
[広告ネットワーク]            [プリランダーLP]            [オファー(アフィリ)]
 クリック発生                 クリックIDを受取り            無料登録が発生
 clickid=ABC を付与  ──→     OFFER_URL?subid=ABC ──→     subid=ABC を記録
        ▲                                                      │
        └──────────  postback: …&extid=ABC  ←───────────────────┘
                    「ABCが成約」と広告側へ返す＝CVが管理画面に1件立つ
```

**直結 vs トラッカー**
- 1オファーだけなら**直結でOK**（Chaturbate SOI直リンクが最も簡単）。学習1周はこれで十分。
- **postbackの全体像を体感したい**なら、間に**無料トラッカー BeMob**（無料枠 **月10万イベント**）を噛ませるのが推奨。広告↔BeMob、オファー↔BeMob の2本のpostbackを自分で設定する＝学びが濃い。Voluum（月$99〜）は不要。

- [ ] **損益分岐CV率 = CPC ÷ 単価** を自分のケースで計算できる

## 7. コンプラ・安全メモ（アカウント停止を踏まないため）

### 7-1. アカウント停止の典型理由と回避
| 典型理由 | 中身 | 回避（初心者の最低ライン） |
|---|---|---|
| **クローキング** | 審査用と実配信用でページを出し分ける | **やらない。** 審査も実配信も同じLPを見せる。学習1周で小細工は不要かつ命取り。 |
| **誤認/誇大クリエイティブ** | 「必ず会える」「無料でヤレる」「あなたの地域の人妻が…」等の嘘・煽り | 事実ベースの無難な文言に。実在人物・有名人の写真や偽の通知UIを使わない。 |
| **年齢ゲート無し** | 18禁ゲートがないLP | 入口に年齢確認ゲート必須（本LPは実装済み）。 |
| **運営者情報なし** | 連絡先・規約・プライバシーが無い | 最低限のフッター情報を置く。 |
| **オファー規約違反のトラフィック** | 禁止された面/手法（incentive, popunder禁止案件にpop 等） | オファー詳細の「禁止事項」を**送客前に必ず読む**。Chaturbate/StripCashのpop可否を確認。 |
| **ブランド/商標の無断使用** | cam各社のロゴ・名称を許可なくクリエイティブに | 自分のニュートラルな素材で。各社のブランド利用規約を確認。 |

### 7-2. 決済・年齢確認まわりの最低限の遵守
- **18禁の明示**と年齢確認ゲートを入口に置く（未成年アクセスを設計で拒否）。
- 送客先は**正規の大手cam（Chaturbate/Stripchat等）**に限定。素性の不明なオファーへ送らない。
- 入金は**自分名義のカード/ウォレット**で。第三者名義・不正な決済手段は使わない（即停止＋資金没収）。
- **児童に関する素材・暗示は一切扱わない**（コンテンツも、クリエイティブも、送客先も）。ここは一発退場かつ法的に重大。
- 露骨な画像はLP/クリエイティブに載せない（本案は調査・設計のみで完結。送客はテキスト＋年齢ゲートで足りる）。
- 【要確認】無料ホスティング（Cloudflare Pages等）の**アダルト/アフィリ規約**。NGなら、アダルト可のホスティングに置く。
