# アダルトアフィリエイト A案（資産型SEO）実行プラン（調査・設計）

> 対象：FANZA/DUGA/ライブチャット/物販グッズ等を、自社サイト（SEO＋SNS）で多点回収する**資産型（ストック型）**モデル。
> スコープ：本書は**調査・設計のみ**。露骨コンテンツ・記事本文は作らない。
> 作成日：2026-06-27 ／ 形式：比較表中心、最後に30日ロードマップ1枚

---

## 0. 資産型モデルの考え方（最初に）

B案（出会い系CPA＝即金フロー型）との最大の違いは、**継続報酬（ストック）で「資産」を積む**点。

- **狙いは「継続報酬」の母数を増やすこと。** DUGAの永久報酬・DXLIVEの継続報酬・FANZA月額が積み上がるほど、後から記事を書かなくても入る収益が増える。
- **初収益までは遅い**（SEOが効くのに通常3〜6ヶ月）。SNS（X）で初期流入を補い、その間に記事と継続報酬の母数を仕込む。
- **片手間前提**なので、量産しても品質が崩れない「型（テンプレ＋チェックリスト）」を先に作るのが肝。

### 2-1. サブジャンル評価マトリクス

> 各項目5点満点（高いほど初心者有利）。**継続報酬との相性**を重視して重み付け。

|---|---|---|---|---|---|---|
| ライブチャット（DXLIVE等） | 4 | 3 | 3 | **5** | 15 | 継続報酬が最強。レビュー資産が効く |
| アダルトグッズ（物販） | 3 | **4** | 4 | 2 | 13 | 競合薄め・エバーグリーン。Amazon併用可 |

| 素人/熟女など嗜好特化 | 4 | 3 | 4 | 4 | 15 | 検索意図が明確で記事化しやすい |

### 2-2. 初心者が最初に1本に絞るべき特化ジャンル 3案

| | 案A：ライブチャット特化 | 案B：アダルトグッズ・レビュー特化 | 案C：特定嗜好の動画レビュー特化（DUGA軸） |
|---|---|---|---|
| **想定読者** | ライブチャットを試したい男性／使い方を知りたい層 | グッズ初心者・カップル・女性／失敗したくない層 | 特定ジャンル（例：素人・熟女等）を深く探す濃いファン |
| **狙う検索意図** | 「ライブチャット おすすめ/比較/無料/使い方/安全」 | 「{商品カテゴリ} おすすめ/使い方/比較/口コミ」 | 「{ジャンル} 動画 おすすめ/作品/レビュー/比較」 |

| **強み** | **継続報酬で資産化が最速**。単価高い | **競合が薄く**エバーグリーン。物販で堅実 | **DUGA永久報酬**と噛み合い、ニッチで上位化しやすい |

## 3. キーワード設計

### 3-1. 検索意図グルーピング（「買う/登録する直前」のワード）

| 意図グループ | 代表ワード（{}は特化ジャンル名を差し込む） | 成約への近さ | 競合 | 初心者優先度 |
|---|---|---|---|---|
| **比較・ランキング** | {ジャンル} おすすめ／比較／ランキング／人気 | 高 | 強 | 中（ロングテール化して狙う） |
| **口コミ・評判** | {サービス} 評判／口コミ／レビュー／実際 | 高 | 中 | **高** |
| **使い方・登録** | {サービス} 使い方／登録方法／無料／始め方 | 高 | 中 | **高** |
| **料金・解約** | {サービス} 料金／値段／解約／退会 | 中〜高 | 弱〜中 | **高**（不安解消で成約後押し） |
| **安全性** | {サービス} 安全／危険／身バレ／本人確認 | 中 | 弱 | **高**（初心者の不安に応える） |
| **比較対象** | {A} vs {B}／{A} と {B} どっち | 高 | 中 | 中 |

### 3-2. ロングテール優先順位の考え方

```
優先度 = 成約への近さ ÷ 競合の強さ
```
- **最初に取りに行く**：`{サービス名}＋口コミ/使い方/料金/安全`（指名×不安解消＝競合が弱く成約近い）
- **次に**：`{ジャンル}＋おすすめ＋{条件}`（例：無料・スマホ・初心者）の**3語以上ロングテール**
- **後回し**：`{ジャンル} おすすめ`単体のビッグワード（競合が強すぎ、ドメインが育ってから）

> 需要・競合の正確な推定は、ラッコキーワード等のサジェスト＋実際の検索結果の上位の強さ（大手か個人か）を目視で判断。数値ツールは導入後に。

## 4. サイト構造設計（多点回収）

### 4-1. 階層設計

```
[トップ] 特化ジャンルのハブ。主要カテゴリと人気記事への導線
  ├─ [カテゴリ] サービス別 / 用途別 / 価格帯別 など
  │     ├─ [比較・ランキング記事] ← 集客と回遊の中心。複数案件を横並べ
  │     │       └─ 各案件の[個別レビュー記事]へ内部リンク
  │     └─ [個別レビュー記事] ← 指名KWで成約。CTAを集中配置
  └─ [お役立ち/ガイド記事] 使い方・安全性・解約 等（不安解消で成約後押し）
```

### 4-2. 多点回収：収益ポイントの配置と規約注意

| 位置 | 置くもの | 規約・運用注意 |
|---|---|---|
| 記事内 本文中CTA | 主役案件（例：レビュー対象のFANZA/DUGA/DXLIVE） | **各ASPのリンクは自社サイト経由が基本**。直リン禁止案件はリンク先・計測規約を遵守 |
| 記事内 比較表 | 複数案件を横並べ（動画＋ライブ＋物販） | 1記事に詰め込みすぎてユーザー体験を壊さない。主役1＋脇役2程度 |
| 関連枠（記事下） | 関連レビュー・関連グッズ（物販） | 物販は薬機法（効能表現NG）に最注意 |
| サイドバー | 人気ランキング・継続報酬系（DXLIVE/DUGA） | 全ページ共通枠で継続案件を露出＝資産の積み上げ |
| フッター | 年齢確認・運営者情報・免責・PR表記 | コンプラ表記は全ページ必須（§7） |

> **多点回収の原則**：1記事＝1主役。脇に「動画↔物販↔ライブ」を**自然に**置いて取りこぼしを拾う。詰め込み過ぎは離脱と規約違反の元。

### 4-3. 技術構成：WordPress vs 静的SSG

| 構成 | 量産/管理 | SEO | 速度 | 保守 | 初心者推奨度 |
|---|---|---|---|---|---|
| **WordPress** | ◎（カテゴリ/タグ/量産・予約投稿が容易） | ◎（プラグインで内部SEO・サイトマップ） | △（要高速化） | △（更新・セキュリティ・アダルト可サーバ必須） | **★★★（資産型はこれ）** |
| **静的SSG（Astro/Hugo）** | △（記事管理は手間／Git運用） | ○ | ◎ | ○ | ★（Claude Code量産と相性は良いが運用が玄人向け） |

> **推奨：WordPress**（アダルト可のレンタルサーバを選ぶこと）。理由は、資産型は**カテゴリ管理・大量記事の運用・内部リンク**が肝で、非エンジニアでも回せるWPが圧倒的に有利。Claude Codeは「記事テンプレHTML生成・内部リンク設計・構造化データ・一括整形」の補助に使うのが現実的。（※B案のLPは静的推奨だったが、資産型サイトはWP推奨で結論が逆になる点に注意）

H2: おすすめ{N}選 比較表  ← 比較表(料金/特徴/こんな人向け/CTA)

H2: 第2位 {サービスB}  → 同上
H2: 第3位 {サービスC}  → 同上

H2: よくある質問（料金/解約/安全性 …不安解消＝成約後押し）

### 5-2. 個別レビュー型 見出しテンプレ

```
H1: {サービス}の口コミ・評判は？使ってわかった{結論}
  リード: 結論（プレースホルダ: {誰}には{おすすめ/不向き}）
H2: {サービス}の基本情報（料金/特徴/運営）  ← 表
H2: 良い口コミ・メリット（{P1}{P2}{P3}）
H2: 悪い口コミ・デメリット（{N1}{N2}）  ← 中立性で信頼獲得
H2: 料金・支払い・解約方法  ← 不安解消
H2: 安全性・身バレ対策・年齢確認  ← 不安解消
H2: {サービス}が向いている人/向かない人
H2: 登録・使い方ステップ  ← CTA集中
H2: よくある質問 / まとめ（CTA）
```

### 5-3. AI量産時の品質チェック項目

- [ ] 結論が冒頭にあるか（検索意図への即答）
- [ ] 一次情報（料金/仕様）が**公式準拠で最新**か（古い料率を書いていないか）
- [ ] デメリットも書き中立性があるか（全肯定は信頼を損ね、誇大表示リスク）
- [ ] §7のNG表現・断定表現が無いか
- [ ] PR表記・年齢確認・運営者情報があるか
- [ ] 内部リンク（比較→個別、個別→比較）が貼られているか
- [ ] タイトル/見出しに狙いKWが自然に入っているか
- [ ] 重複・薄い内容の量産になっていないか（1記事1検索意図）

## 6. SNS集客設計（X／オーガニック）

### 6-1. アダルトジャンルでのX規約準拠（必須設定）

| 項目 | 設定 | 根拠/注意 |
|---|---|---|
| センシティブ設定 | 「あなたのポスト」を**センシティブ設定ON** | アダルト投稿には必須。怠ると凍結リスク |
| 年齢/配信対象 | 成人向け性的コンテンツは**配信対象21歳以上**設定（2025年7月規定） | 規約改定に対応 |
| アカウント種別 | アダルト専用アカウントとして運用（本垢と分離） | 監視が厳しく、違反で即凍結 |
| 投稿内容 | 盗撮・本人同意のない私的画像は**絶対NG** | 一発凍結＋法的リスク |
| 送客リンク | プロフ/固定ポストに自社サイト導線 | 2025年10月、アフィリ送客サイトの**広告は認可条件付きで解禁**（オーガニックは従来通り運用可） |

### 6-2. SEOが効くまでの初期流入を補う運用フロー

| フェーズ | SNSの役割 | 頻度（片手間想定） | 自動化できる部分 |
|---|---|---|---|
| 0〜1ヶ月 | アカウント育成・サンプル投稿で関心層を集める | 1日1〜3投稿 | 予約投稿（Buffer等）／作品リンク自動変換系ツール |
| 1〜3ヶ月 | 記事公開のたびに告知、反応を見てKW着想 | 記事公開時＋日次1〜2 | 新記事の自動告知（RSS連携） |
| 3ヶ月〜 | SEO流入が立ち上がるまでの下支え・指名検索の喚起 | 日次1〜2（無理しない） | 過去投稿の再ポスト自動化 |

> **片手間で回すコツ**：投稿は「予約でまとめ作成」、リンクは「自動変換ツール」、告知は「RSS自動連携」。手作業は反応の良い投稿への返信程度に絞る。凍結に備え**サイトを母艦**にし、SNSは流入の一手段と割り切る（SNS依存は禁物）。

## 7. コンプラ・チェックリスト（サイト・記事向け）

### 7-1. 法令別チェックリスト

**A. 薬機法（物販グッズで最重要）**

- [ ] グッズに**医薬品的な効能効果**をうたっていない（「治る」「増大」「不感症改善」「精力増強」等NG）
- [ ] 「健康になる」「体質改善」等の身体への作用を断定していない
- [ ] あくまで**製品の仕様・使用感**の範囲で記述（効能ではなく特徴）

**B. 景品表示法（PR表記・誇大/断定）**

- [ ] 全記事/全ページに**「広告(PR/アフィリエイト)を含む」表記**（ステマ規制対応）
- [ ] 「必ず」「絶対」「100%満足」等の**断定・保証表現**なし
- [ ] 「日本一」「No.1」等は**客観的根拠**がなければ使わない
- [ ] 「最安」「今だけ」等の価格・期間表記は事実と一致（有利誤認回避）
- [ ] 口コミ・レビューは捏造でない（体験談の実在性）

**C. 年齢確認・ゾーニング（必須要件）**

- [ ] サイト入口に**18禁・年齢確認**（R18ゲート/確認ボタン）
- [ ] **「18歳未満閲覧禁止」**を明示（フッター等）
- [ ] 児童（18歳未満）を想起させる表現・素材は一切NG（重大）
- [ ] アダルト可のサーバ/ドメインで運用（規約違反でのBAN回避）

### 7-2. 「使ってはいけない表現」具体例リスト

| 区分 | NG表現の例 | 理由 |
|---|---|---|
| 薬機法（効能） | 「精力増強」「増大する」「不感症が治る」「ED改善」「ホルモン分泌」 | 医薬品的効能＝薬機法違反 |
| 断定・保証 | 「必ず満足」「絶対イケる」「100%効果あり」 | 優良誤認（景表法） |
| 誇大・No.1 | 「日本一気持ちいい」「業界最強」「全員リピート」 | 根拠なき優良誤認 |
| 有利誤認 | 「今だけ無料」（恒常的に無料）「最安値」（裏取りなし） | 有利誤認 |
| 児童関連 | 「JK」「学生」「○歳」など未成年想起、未成年を肯定的に扱う一切 | 児童保護・重大違反 |
| 露骨/品位 | 過度に露骨な性的描写を広告/タイトルに | 各プラットフォーム規約違反 |

> **言い換えの方向**：効能→使用感・仕様（「刺激の強さ」「静音設計」等の事実）、保証→個人の感想（「人によって感じ方は異なります」）。具体コピーは本書スコープ外。
