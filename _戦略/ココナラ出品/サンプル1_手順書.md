# 設定手順書 — フォーム回答の自動通知

所要時間: 約10分 / プログラミング知識は不要です

## できること

Googleフォームに回答が届いた瞬間、指定のメールアドレス(と、必要ならSlack)に回答内容が自動で通知されます。

## 手順

### 1. スクリプトを貼り付ける

1. フォームの回答が記録されるスプレッドシートを開きます
2. メニューの「拡張機能」→「Apps Script」をクリックします
3. 開いた画面に元からあるコードをすべて消し、納品した「サンプル1_フォーム通知.gs」の中身を貼り付けます
4. 上部の保存アイコン(フロッピー)をクリックします

### 2. 設定を書き換える

コード上部の `CONFIG` の中を書き換えます。

- `MAIL_TO`: 通知を受け取るメールアドレス。複数の場合は `'a@example.com,b@example.com'` のようにカンマ区切り
- `SLACK_WEBHOOK_URL`: Slack通知を使う場合のみ、Incoming WebhookのURLを貼り付け(使わなければそのまま)
- `EXCLUDE_FIELDS`: 通知に含めたくない項目名。例: `['タイムスタンプ']`

### 3. テスト送信する

1. 画面上部の関数選択プルダウンで `testNotify` を選び「実行」をクリックします
2. 初回は「承認が必要です」と表示されます。自分のGoogleアカウントを選び、「詳細」→「(プロジェクト名)に移動」→「許可」の順に進みます
   ※「このアプリはGoogleで確認されていません」という警告は、自作スクリプトでは必ず表示されるものです
3. `MAIL_TO` のアドレスにテストメールが届けば成功です

### 4. 自動実行を有効にする

1. 関数選択プルダウンで `setupTrigger` を選び「実行」をクリックします
2. フォームから実際にテスト回答を1件送信し、通知が届くことを確認します

以上で設定完了です。

## うまく動かないとき

- **メールが届かない**: 迷惑メールフォルダを確認してください。`MAIL_TO` の綴りも確認してください
- **「承認が必要です」から進めない**: 会社のGoogle Workspaceでは管理者がスクリプトを制限している場合があります。管理者にご確認ください
- **回答は記録されるのに通知が来ない**: `setupTrigger` を実行し忘れている可能性があります。手順4をやり直してください
