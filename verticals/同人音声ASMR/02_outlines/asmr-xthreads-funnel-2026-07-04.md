# X＋Threads ファネル設計｜2026-07-04

SNSから受け皿を経てDLsiteへつなぐ導線の設計です。
前提: [[asmr-platform-rules-2026-07-04]] / [[asmr-hybrid-strategy-2026-07-04]]。
方針は全年齢で集客し、年齢認証を挟んでR18収益へ橋渡しします。

## 1. 役割分担(3層)

各面の役割を分けます。1つに全部を負わせません。
- Threads(非露骨): 全年齢の癒し・睡眠で広く接触。間口づくり。
- X(踏み込み可): 全年齢アカで発見、R18アカでサンプルと収益前段。
- 受け皿(ブログ/LP): アフィリを集約し、CVを取る中心。

## 2. 導線図(全年齢→R18)

```
Threads(全年齢/癒し) ─┐
                      ├→ プロフ1リンク → 受け皿(全年齢まとめ) → 年齢認証 → R18レビュー → DLsite
X 全年齢アカ ─────────┘                                   ↑
X R18アカ(サンプル/年齢認証つき) ───────────────────────────┘
```

- Threadsは本文リンクを避け、プロフィール1リンクへ集約します。
- Xは全年齢アカが発見、R18アカが年齢認証つきで作品へ送ります。
- 受け皿の全年齢ページとR18ページはURL/カテゴリで分離します。

## 3. 各段の狙いと指標

- 接触(SNS): フォロワー・インプレ・保存。フックは [[asmr-winning-hooks-2026-07-04]]。
- 遷移(受け皿へ): プロフィールリンク率・クリック数。
- 収益(DLsite): アフィリCV・単価。基準は [[asmr-success-criteria-2026-07-04]]。

## 4. ゾーニングと表記(必須)

- R18リンクの前に年齢確認(18歳以上)を必ず挟みます。
- 全年齢アカからR18への直リンクはしません。認証導線を経由します。
- 投稿には「PR」「広告」を明記します。誤誘導・射幸はしません。

## 5. リスク分散

- 凍結・削除に備え、受け皿(独立ブログ/LP)を資産の中心にします。
- SNSは送客チャネルと位置づけ、単体依存を避けます。
- アフィリは受け皿に集約し、SNS本文の直アフィリを最小化します。
