# ポケカ相場ブログ 自動運用｜導入検討と設計（2026-07）

X とブログ併用の運用を、どこまで自動化できるかを調べ、導入可否を判断し、導入する場合の最重要ポイントを設計しました。既存の [[RULES_division_and_automation]] と [[COWORK_設計図v2_アダルトアフィリ自動化]] の方針を、ポケカに翻案しています。

---

## 結論（先に）

- **導入は可能**です。ただし全自動ではなく「**半自動**」です。
- 自動化の上限は既存ルールどおり「**WordPress下書き保存まで／X投稿は予約まで**」。**公開と相場判断は人間**が握ります。
- **全自動（公開まで無人）は非推奨かつ危険**です。理由は、SEOとXの価値の源泉（一次データ・相場観・実体験）が、まさに自動化できない部分だからです。ここを自動化すると価値がゼロになります。

---

## 何が自動化できるか（可否マップ）

| 工程 | 自動化 | 手段 | 根拠・注意 |
|---|---|---|---|
| ネタ出し・リサーチ | ○ | Coworkスケジュールタスク（週次でClaudeが実行→vaultに出力） | ファイル出力で止める |
| 記事の下書き生成 | ○ | 同上（DRAFTエージェント相当） | 【要検証】【リンク挿入】を必ず埋め込む |
| X投稿文の一括生成 | ○ | 同上（週7本→ファイル） | AI生成はX規約OK |
| WordPressへ下書き投入 | ○ | WP REST API＋アプリケーションパスワード（status=draft） | 下書きのみ。公開はしない |
| X投稿の予約 | △（半自動） | 認可ツール（純正予約／Buffer／Typefully等） | 自作投稿の予約はOK |
| 定点観測の価格収集 | △（半自動） | 人間が値をコピー→Claudeが整形 | 相場サイトのスクレイピングはToSリスク |
| 相場判断・一次データ | ✕ | 人間 | 差別化＝収益とSEOの源泉。自動化しない |
| 事実確定（【要検証】） | ✕ | 人間 | 価格・日付・発行数の裏取り |
| 記事の公開・リンク設置 | ✕ | 人間 | 取り消せない操作。関所を残す |
| X の いいね/フォロー/リプ | ✕ | 手動のみ | **自動エンゲージは規約違反・BAN対象** |

---

## 技術的手段の詳細

**1. Coworkスケジュールタスク（生成の自動化）**
週次でClaudeを定期実行し、リサーチ・記事下書き・X投稿案を vault に出力できます。出力までで止め、人間が確認します。既存の同人音声パイプライン（`03_drafts/_生成ログ.md`）と同じ発想です。

**2. WordPress REST API（下書き投入の自動化）**
WordPress 5.6以降の「アプリケーションパスワード」（24文字トークン）で、外部から `/wp-json/wp/v2/posts` に `status=draft` で投稿を作れます。**下書き保存までは無人で可能**、公開は人間がボタンを押します。ルールの「ワンクリックで公開できる状態まで」を技術的に満たせます。

**3. X の投稿自動化（予約はOK・エンゲージはNG）**
- **自作投稿の予約**は、認可ツール（純正予約・Buffer・Typefully・Hootsuite等、公式OAuth経由）で完全に許可されています。AI生成文の投稿もOK。
- **禁止**: いいね・フォロー・リプ・DMの自動化、キーワード自動リプ、複数アカウントで酷似投稿。2026年は取り締まりが強化され、**OAuthを迂回するブラウザ自動化（自動ログイン等）はBAN対象**です。→ Xはツール予約に留め、交流は手動で行います。
- **X API直投稿はコスト高**: 2026年2月から従量制。1投稿$0.015、**リンク付きは$0.20**、レート制限あり。→ APIで自動投稿するより、予約ツール＋手動が現実的。リンクはそもそも本文に貼らない方針（[[pokeca-運用ナレッジ_XとブログHUB_2026-07]]）とも整合。

**4. 定点観測（半自動が安全）**
相場サイトの自動スクレイピングは各サイトのToS・負荷の観点でリスクがあります。人間が「売れた価格」を数枚コピー→Claudeがトラッカー整形・グラフ化・所感ドラフト、が安全で速い形です。

## 導入できるか（検討）

**判定: 導入可能（半自動）。** 3つの理由です。

1. **手段が全て揃っている**: スケジュールタスク＋WP REST API＋X認可予約ツール。専用コネクタは不要（登録上も該当コネクタは0件）。
2. **既存ルールと完全一致**: 「下書きまで自動・公開は人間」の思想をそのまま適用できる。新たな判断が要らない。
3. **リスクが管理可能**: 危険な操作（公開・エンゲージ・ブラウザ自動化）を最初から除外すれば、BAN・誤公開の芽を断てる。

**制約（承知して進める点）**:
- X API直投稿は有料化。→ 予約ツール＋手動で回避。
- 相場データの自動収集はToSリスク。→ 半自動。
- **AIそのままの薄い記事はSEOで沈む**。→ 差別化レイヤーは人間必須。

---

## 【最重要】導入する場合に守る核

自動運用を入れると、パイプラインが運用の背骨になります。だからこそ、次の4点が**最重要のガードレール**になります。ここが崩れると、自動化はむしろ事業を壊します。

1. **人間の関所を1つ残す**: 公開・アフィリリンク設置・【要検証】の確定は人間。取り消せない操作は自動化しない。
2. **差別化レイヤーは自動化しない**: 相場判断・一次データ（定点観測）・実体験・物語。ここがSEO（E-E-A-T・情報の独自性）とXの価値の源泉。AIは調査・構成・初稿まで、価値は人間が上書きする。
3. **X自動エンゲージ禁止・OAuth迂回禁止**: いいね/フォロー/リプの自動化とブラウザ自動ログインはBAN直結。予約のみ、交流は手動。
4. **リンクをAPIで自動投稿しない**: コスト（$0.20/投稿）と到達の両面で不利。本文リンクなし・誘導は手動が原則。

> ひとことで: **「作る」は自動、「決める・出す・関わる」は人間**。この線を最重要に据えます。

## 導入アーキテクチャ（ポケカ版パイプライン）

既存のDispatch編成をポケカに翻案します。

```
週次スケジュールタスク（Cowork・Claude）:
  IDEA     : 今週の柱記事案＋X7本のテーマを提案（30周年カレンダー反映）
  RESEARCH : 選んだテーマの検索意図・KW・競合を調査 → 01_research/
  DRAFT    : 記事下書き生成（【要検証】【リンク挿入】【内部リンク】埋め込み）→ 03_drafts/
  X-BATCH  : 単体完結のX投稿7本を生成（本文リンクなし）→ ファイル
  （任意）WP-DRAFT : 合格下書きをWordPressに status=draft で投入
  → サマリ報告：完成本数／要確認本数

人間（松岡）が握る関所:
  - 定点観測の価格入力（半自動の起点）
  - 相場判断・実体験の上書き
  - 【要検証】の裏取り
  - アフィリリンク設置・記事の公開
  - Xの予約セット＋交流（いいね/リプ）
```

## 段階導入ロードマップ

- **Phase 0（今）**: 手動で1〜2週まわし、週次オペ（[[pokeca-運用設計_週次オペレーション]]）を体に入れる。
- **Phase 1**: スケジュールタスクで「週次リサーチ＋記事下書き＋X7本」の**生成だけ**を自動化（ファイル出力で止める）。
- **Phase 2**: WordPress にアプリケーションパスワードを設定し、**下書き自動投入**を追加。
- **Phase 3**: 生成品質が安定したら、X予約ツールを連携し「予約セットまで」を効率化。エンゲージは手動のまま。

各フェーズとも、公開・リンク・相場判断の関所は人間に残します。

## 次に決めること
