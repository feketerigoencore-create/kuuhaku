# ポケカ動画 → DMMマイカ在庫登録 運用手順（確立日: 2026-07-21）

スマホで撮ったカード動画から、PC上のDMMマイカ（BlueStacks内アプリ）へPSA在庫を自動登録する手順。
※原本: G:\マイドライブ\ポケカ在庫登録_運用手順.md（Cowork用）。こちらはVault保管用コピー。

## 毎回のフロー（ユーザー作業は2ステップ）

1. スマホで動画撮影 → Googleドライブ「マイドライブ」直下にアップロード
2. Cowork（新しいチャット推奨）で「ドライブの動画を取り込んでマイカに登録して」と依頼

以降はClaudeが実行:

1. G:\マイドライブ の動画を確認（※G:はbashサンドボックスにマウント不可。エクスプローラー操作かダウンロードフォルダ経由でコピー。前回はDownloadsに実体があった）
2. ffmpegでフレーム抽出: `ffmpeg -i IMG_XXXX.MOV -vf "fps=1,scale=960:-1" frames/XXXX_%02d.jpg`
3. フレームをReadで確認し、カード名・型番・レアリティ・PSAグレード・値札を識別
4. 識別結果の一覧をユーザーに提示して登録対象を確認
5. BlueStacks内のDMMマイカで登録（下記の操作ノウハウ参照）
6. マイカードの枚数増分で反映を検証

## 環境（構築済み）

- BlueStacks 5（PC、Googleログイン済み）+ DMMマイカアプリ（ログイン済み）
- マイカはアプリ専用。Web版は存在しない（myca.cards=紹介LP、myca.dmm.com=通販モール）
- 登録規約: フラグ=PSA、ランク=10、購入金額=店頭値札、枚数=1

## マイカ操作ノウハウ（重要・ハマりどころ）

- **日本語入力**: typeでは入らない。`write_clipboard` → `ctrl+v` で貼り付ける
- **検索**: 貼り付け後のReturnで反映されないことが多い。検索欄を再クリック→Returnをもう一度。ただし検索欄の位置は状態で変わる（一覧表示時 y≈181 / フォーカス時 y≈111）。フォーカス時に y≈181 をクリックすると1枚目のカードを開いてしまうので注意
- **検索語**: 「&」は全角「＆」で（マオ＆スイレン等）。半角&は0件になる
- **検索欄のクリア**: ctrl+a が効かない。End → Backspace×30〜40
- **フラグ/ランクのドロップダウン**: 1回目の「開いて選択」はほぼ失敗する。開く→2秒待つ→選択、を選択できるまで繰り返し。zoomで PSA / 10 になったのを確認してから登録ボタン
- **スクロール**: ホイールは効かないことが多い。`left_click_drag`（650→250）でドラッグスクロール
- **誤タップ注意**: ＋ボタンを誤クリックすると即1枚登録される。気づいたら−で取り消し。作業後は必ずマイカードの増分（枚数）で検証する
- **登録フォーム座標（960px表示時の目安）**: 購入金額(711,311) / 登録枚数(711,374) / フラグ(611,437) / ランク(812,437) / 登録(711,745)

## 撮影のコツ（トークン節約・認識精度向上）

- 1枚あたり約2秒、PSAラベルと値札が正面でブレなく映るように
- 1回の購入分は1本の動画にまとめる
- 依頼は毎回新しいチャットで（履歴が長いと毎回その分を読み込んで高コスト）

## v2 最小トークン実行計画（次回はこの方式を優先）

1. **一覧画像方式**: フレームを個別に見ず、タイル合成した1〜2枚だけ見る
   `ffmpeg -i IN.MOV -vf "fps=1,scale=480:-1,tile=4x3" sheet_%d.jpg`
   判読できないタイルだけ元フレームをzoom/Readで確認
2. **OCR先行**: `tesseract frame.jpg out -l jpn` で値札・PSAラベルをテキスト化し、画像閲覧を減らす（要: `apt/pip`でtesseract導入、初回のみ）
3. **ブラインド登録**: カード1枚につき「clipboard書込 → 検索〜登録を座標で一括実行（スクショなし、フラグ/ランク選択サイクルは2回実行して確実化）→ 最後にzoom1回で枚数確認」。失敗時のみフルスクショ
4. **検証1回**: 全カード登録後にマイカードの合計枚数増分をzoomで1回確認するだけ
5. （検討）スマホのマイカアプリに**カメラでカード検索/スキャン機能**があれば、そもそもPC経由不要でトークンゼロ。一度アプリ内を確認する価値あり

## Claude向け・トークン節約ルール（実行時に守る）

- この手順書を最初に読み、探索や試行錯誤をやり直さない（Web版を探す・経路を調べ直す等は不要）
- フレームは全部見ない。カード数×2枚目安でサンプリングし（例: 2秒/枚撮影なら奇数フレームのみ）、判読できたカードのフレームは追加で見ない
- 画面確認はフルスクリーンショットでなく`zoom`（対象領域のみ）を優先。登録成功確認もzoomで枚数表示だけ見る
- 検索→登録は本書の座標・ノウハウ通りに1回で決める。失敗時のみスクリーンショット
- 最終検証はマイカード合計枚数の増分1回で済ませ、個別の再スクロール確認は枚数が合わない時だけ行う

## 制約（ユーザーにしかできないこと）

- 各種ログイン（Google/DMM/マイカ）のパスワード入力
- 外部からダウンロードしたexeの実行（インストーラ起動）とUAC承認
- ブラウザからのファイルダウンロードは拡張機能経由だとブロックされる → ユーザーがクリック
