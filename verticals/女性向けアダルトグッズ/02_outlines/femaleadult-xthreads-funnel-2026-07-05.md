# X＋Threads ファネル設計｜2026-07-05

SNS から受け皿を経て ASP の成約に至る導線を設計します。
Threads は非露骨・本文リンク不可、X はやや踏み込み可、という役割差を前提にします。
成約は必ず受け皿(自分の資産)で取ります。SNS単体アフィリに依存しません。

## 0. 全体像(3層)

認知(SNS) → 送客(プロフ/リンク) → 成約(受け皿→ASP)。

```
[Threads] 共感・会話 ──┐
                       ├─→ プロフ/固定/自己リプのリンク ─→ [最小LP/比較記事] ─→ [ASP: NLS等]
[X] Tips・比較の要点 ──┘
```

## 1. 媒体の役割分担

### Threads(やわらかい入口)
- 役割: 非露骨の共感・会話でフォロワーの裾野を広げる。
- 出す型: お悩み共感/一次の声要約/場面提案/問いかけ。
- リンク: 本文NG。プロフ固定＋自己リプ、露出は週3まで。
- 目的: 「信頼」と「プロフ誘導」。直接成約は狙わない。

### X(踏み込む集客)
- 役割: Tips・比較の要点で購入検討層を受け皿へ送る。
- 出す型: 選び方Tips/比較の要点/チェックリスト/失敗回避。
- リンク: 投稿末・返信で受け皿へ(露骨化しない)。
- 目的: 「送客」。検討層をLP/比較記事へ流す。

## 2. 受け皿(コンバージョンの中心)

- 最小LP 1枚＋比較記事3本を先に用意します(詳細は別ファイル)。
- 受け皿でPR表記・R18年齢確認・医療非断定・一次の声を満たします。
- SNSは「借り物」。凍結に備え、必ず受け皿へ送って資産化します。

## 3. 各段のKPIと閾値

| 段階 | 媒体 | KPI | 見る頻度 |
|---|---|---|---|
| 認知 | X/Threads | フォロワー・インプレ | 週次 |
| 関心 | X/Threads | 保存・返信・プロフ表示 | 週次 |
| 送客 | プロフ/リンク | 受け皿クリック数 | 週次 |
| 成約 | 受け皿/ASP | CV(アフィリ発生) | 月次 |

判定ラインは [[femaleadult-success-criteria-2026-07-05]] に準拠します。

## 4. 導線の定型(コピー雛形)

- Threads本文: 「詳しい選び方はプロフィールのリンクにまとめています。」
- Threads自己リプ: 「(補足)比較の詳細はこちら → LP」。
- X投稿末: 「選び方の要点はこちらに整理しました → LP(PR)」。
- 受け皿CTA: 「気になる方は各公式で最新の仕様・価格を確認できます(PR)。」

## 5. リスクと分散

- Meta/X の規約改定・凍結を前提に、受け皿へ必ず資産を逃がす。
- ASP は主力2〜3社(NLS＋バナーブリッジ等)に絞る。
- 露骨化しない。露骨度を上げると凍結・審査落ちで導線ごと崩れる。

## 6. 要確認
- Threads の自己リプ・プロフリンクの現行許容範囲。
- 受け皿→ASP のCV計測方法(クリック計測・パラメータ設計)。
