# AffiliateAuto 下ごしらえテンプレート（Cowork用）

このファイルは、アフィリエイト記事の「リサーチ→構成→下書き」を
ジャンル・ニッチを差し替えるだけで回すための指示書です。
Cowork のプロジェクトに置き、先頭のパラメータだけ書き換えて使います。

人間（あなた）の担当：パラメータ設定／ニッチ最終選定／構成GO判断／実体験の提供（必要な型のみ）／WordPress投稿・リンク設置
Claude の担当：下記フローの①②③（下ごしらえ）

====================================================
■ パラメータ（毎回ここだけ書き換える）
====================================================
- ジャンル：________（例：成人向けコミック）
- 記事タイプ：________（レビュー型／まとめ・キュレーション型／ハウツー型 から選ぶ）
- ニッチ仮説：________（未定なら「未定」。①の後に人間が確定）
- ターゲット読者：________（任意。分かっていれば書く）
- 実体験の有無：________（あり／なし。「なし」の場合レビュー型は不可、まとめ型を選ぶ）

====================================================
■ 固定ルール（全ジャンル共通・書き換え不要）
====================================================
- 性的に露骨な描写はしない。構造・リサーチ・構成・集客設計が中心。
- 他サイトの文章・画像・動画・紹介文の転載は禁止。すべて自分の言葉で。
- 不確実な情報（検索ボリューム、商品名、品番、価格、報酬単価など）は必ず「要検証」と明記。
- アダルト系はキーワードツールが数値を返しにくい。定量データより構造的な見立てを優先。
- 実体験のないレビューは書かない（捏造禁止）。他人のレビューの寄せ集めもしない。
  → 「他ユーザーの評価傾向」を事実として要約するのは可。ただし記事の主役にはしない。
- 差別化は「対象の優劣」ではなく「切り口・分類軸」で作る。
- 出力は過度な装飾を避ける。表・リストは必要な時だけ。推定箇所は明記。
- 保存先（Cowork上）：リサーチ→01_research／構成→02_outlines／下書き→03_drafts

====================================================
■ フロー①：リサーチ（Claude担当 → 01_research）
====================================================
以下の5点を、上記パラメータのジャンル・ニッチについて出力する。
数値は要検証扱い。「意図の濃さ」と「競合の手薄さ」で優先度をつける。

1. 検索意図の地図：対象読者の検索行動を購買意欲の段階（A情報収集／B比較検討／C購入直前）で整理。
2. 刺さる切り口：読者の本音の悩み・ハードルを抽出。記事の主役はこの悩み解決。
3. 対象カテゴリ／作品群の見立て：紹介しやすい括りを構造で整理（具体名は要検証）。
4. 競合・コンテンツの空き：既出の記事型と、手薄な切り口を特定。差別化軸を3つ出す。
5. 集客・収益動線の仮説：入口→本命→販売の内部リンク設計とCTA位置の当たり。
最後に「次に絞るニッチ候補」を3つ提示し、人間の選定を待つ。
→ 【人間の承認ポイント①】ニッチの最終選定

====================================================
■ フロー②：記事構成案（Claude担当 → 02_outlines）
====================================================
確定ニッチについて、以下を出力する。
- ターゲット読者像（1人に絞る）
- タイトル案3つ（要検証：競合並び・CTRは公開前確認）
- 見出し構成（リード＋H2／H3。各見出しの「狙い」を併記）
- 内部リンク設計（サイト全体での位置づけ）
- トーン＆禁止事項（下書きへの申し送り）
- 要検証リスト
記事タイプに応じて軸を変える：
  - レビュー型：実体験を核に。周辺を他ユーザー傾向で補強。
  - まとめ型：分類の切り口を主役に。視聴・購入を前提にしない。
  - ハウツー型：悩み解決の手順を主役に。商品は受け皿。
→ 【人間の承認ポイント②】構成のGO判断

====================================================
■ フロー③：記事下書き（Claude担当 → 03_drafts）
====================================================
GO後の構成案に沿って本文を執筆する。
- 構成案の見出し順をそのまま本文化。
- 本文中にマーカーを残す：
  【要検証】＝人間がASP等で裏取りする箇所
  【リンク挿入】＝アフィリエイトリンクを貼る位置
  【内部リンク】＝他記事への導線
- 体験談の捏造はしない。実体験を入れる型なら、人間が出したメモを整える形にする。
- 末尾に「公開前チェック（申し送り）」を付ける。
→ 【人間の作業】WordPress投稿・リンク設置・公開判断

====================================================
■ 使い方（Coworkでの回し方）
====================================================
1. このファイルをCoworkプロジェクトに置く。
2. 冒頭パラメータを埋めて、Claudeに「①リサーチを実行」と指示。
3. 出力を見て、人間がニッチを確定（承認①）。
4. 「②構成案を実行」と指示 → GO判断（承認②）。
5. 「③下書きを実行」と指示 → 下書き受領。
6. 人間が要検証箇所を確認し、リンクを設置して投稿。
※ Dispatchで定期実行する場合は、まとめ型（新作キュレーション等）が相性良。
  「毎週、指定ジャンルの新着を①の観点で洗い直す」といったタスク化が可能。
  ※Dispatchの具体的な設定手順は、最新仕様を確認のうえ別途。

====================================================
■ 自動化できる部分・できない部分（今日の結論）
====================================================
自動化できる：リサーチ／構成／下書き／他ユーザー傾向の要約
自動化できない（人間の最小コスト）：
  - 実体験レビューの核（レビュー型を選ぶ場合のみ発生）
  - ニッチ確定・構成GO・公開の各判断
  → 人間のコストを最小にしたいなら「まとめ型／ハウツー型」を選ぶと
    実体験コストがゼロになり、判断と投稿だけが人間の作業になる。

====================================================
■ 分担と自動化範囲（複数環境で回す場合）
====================================================
詳細は別ファイル「RULES_division_and_automation.md」を参照。要点のみ:

役割：
- 企画・下ごしらえ側：リサーチ→構成→下書きを作る（成果物は下書きまで）。
- 投稿準備側：下書きをWordPressに載せる形に整える（成果物はWordPress下書き保存まで）。
- 人間：ニッチ確定／構成GO／要検証の裏取り／リンク設置／公開ボタン。

自動化のゴールは「ワンクリックで公開できる状態まで」。
公開ボタンとアフィリエイトリンク設置は自動化禁止＝必ず人間が手で行う。
公開前の人間チェックという関所を必ず1つ残す（アダルト系は規約違反で
アカウント停止に直結するため）。
